Why not just use OLS?

Let’s plot the binary outcome against liberalism and fit a linear regression:

plot(propetit ~ liberal2, mydata, xlim = c(0, 100), ylim = c(0, 1),
     xlab = "Liberalism", ylab = "Observed Petition Outcome")

lmod <- lm(propetit ~ liberal2, mydata)
abline(lmod)
sumary(lmod)
              Estimate Std. Error t value  Pr(>|t|)
(Intercept) -0.1702657  0.1190660 -1.4300    0.1556
liberal2     0.0138063  0.0024788  5.5697 1.917e-07

n = 109, p = 2, Residual SE = 0.44207, R-Squared = 0.22

This is the linear probability model (LPM). The line goes straight through the data, but notice the problem: for very low or very high values of liberalism, the predicted probability can go below 0 or above 1. That’s nonsensical for a probability. We need a model that constrains predictions to the [0, 1] interval.

Part 2: Logit and Probit Models

Both logit and probit solve the LPM’s problem by passing the linear predictor (\(X\beta\)) through a nonlinear function that maps any real number onto (0, 1):

  • Logit uses the logistic function: \(P(Y=1) = \frac{e^{X\beta}}{1 + e^{X\beta}}\)
  • Probit uses the cumulative normal: \(P(Y=1) = \Phi(X\beta)\)
logitmod  <- glm(propetit ~ liberal2, family = binomial(link = logit), data = mydata)
probitmod <- glm(propetit ~ liberal2, family = binomial(link = probit), data = mydata)

Part 4: Visualizing Predictions

Fitted values vs. the original data

The predict() function can return predictions on two scales: the link scale (log-odds for logit, z-scores for probit) and the response scale (probabilities).

logit.phat2  <- predict(logitmod)                  # Link scale (log-odds)
probit.phat2 <- predict(probitmod)                  # Link scale (z-scores)
logit.phat3  <- predict(logitmod, type = "response")  # Probability scale
probit.phat3 <- predict(probitmod, type = "response") # Probability scale

On the link scale, the relationship is linear by construction:

plot(logit.phat2 ~ mydata$liberal2, lty = 1, type = "l",
     xlab = "Liberalism", ylab = "Predicted Log Odds")
plot(probit.phat2 ~ mydata$liberal2, lty = 2, type = "l", col = "blue",
     xlab = "Liberalism", ylab = "Predicted Z-Score")

On the probability scale, we see the characteristic S-curve:

plot(logit.phat3 ~ mydata$liberal2, lty = 1, type = "l",
     xlab = "Liberalism", ylab = "Predicted Probability")
plot(probit.phat3 ~ mydata$liberal2, lty = 2, type = "l", col = "blue",
     xlab = "Liberalism", ylab = "Predicted Probability")

Overlaying logit and probit curves on the data

Part 5: From Log-Odds to Probabilities Step by Step

This section walks through the math of transforming predictions, which is useful for building intuition about what the logit model is actually doing.

# Start with log-odds predictions
plot(logit.phat2 ~ mydata$liberal2, lty = 1, type = "l",
     xlab = "Liberalism", ylab = "Predicted Log Odds",
     main = "Step 1: Log-Odds (hard to interpret)")

Exponentiate to get odds (\(e^{\text{log-odds}}\)):

plot(exp(logit.phat2) ~ mydata$liberal2, lty = 2, type = "l",
     xlab = "Liberalism", ylab = "Predicted Odds",
     main = "Step 2: Odds (better, but still unintuitive)")

Convert odds to probabilities using \(p = \frac{\text{odds}}{1 + \text{odds}}\):

plot((exp(logit.phat2) / (1 + exp(logit.phat2))) ~ mydata$liberal2,
     lty = 2, type = "l",
     xlab = "Liberalism", ylab = "Predicted Probabilities",
     main = "Step 3: Probabilities (most intuitive)")

This is identical to calling predict(logitmod, type = "response"):

plot(logit.phat3 ~ mydata$liberal2, lty = 2, type = "l",
     xlab = "Liberalism", ylab = "Predicted Probabilities (response)",
     main = "Verification: same curve")

Residuals for binary models

OLS-style residual plots don’t work well for binary outcomes, but we can still examine them. Response residuals are simply \(y - \hat{p}\); Pearson residuals standardize by the variance:

resid.res <- residuals(logitmod, type = "response")
resid.mod <- residuals(logitmod, type = "pear")

plot(resid.res ~ mydata$liberal2, main = "Response Residuals")
plot(resid.mod ~ mydata$liberal2, main = "Pearson Residuals")

The banding pattern (two horizontal strips) is normal for binary outcomes — it’s inherent in the data, not a sign of model failure.

Part 7: Another Example — Cars and Fuel Efficiency

To reinforce these ideas, let’s try a completely different application using the built-in mtcars dataset. We’ll create a binary variable for whether a car has “high” fuel efficiency (above the median MPG):

data(mtcars)
mtcars$high_mpg <- ifelse(mtcars$mpg >= median(mtcars$mpg), 1, 0)

model <- glm(high_mpg ~ wt + hp + am,
             family = binomial(link = "logit"), data = mtcars)

Now let’s generate predicted probabilities across the range of vehicle weight, separately for automatic and manual transmissions:

wt_range   <- seq(min(mtcars$wt), max(mtcars$wt), length.out = 100)
x.auto     <- data.frame(wt = wt_range, hp = mean(mtcars$hp), am = 0)
x.manual   <- data.frame(wt = wt_range, hp = mean(mtcars$hp), am = 1)

prob.auto   <- predict(model, newdata = x.auto, type = "response")
prob.manual <- predict(model, newdata = x.manual, type = "response")

plot_data <- data.frame(Weight = wt_range,
                        Automatic = prob.auto,
                        Manual = prob.manual)

library(reshape2)
plot_data_melt <- melt(plot_data, id.vars = "Weight",
                       variable.name = "Transmission",
                       value.name = "Probability")

library(ggplot2)
ggplot(plot_data_melt, aes(x = Weight, y = Probability, color = Transmission)) +
  geom_line(linewidth = 1.2) +
  xlab("Vehicle Weight (1000 lbs)") +
  ylab("Predicted Probability of High MPG") +
  scale_color_manual(values = c("blue", "red"),
                     labels = c("Automatic", "Manual")) +
  theme_minimal()

As weight increases, the probability of high fuel efficiency drops sharply — and manual transmissions maintain higher probabilities at every weight level.

Part 8: Model Diagnostics

With binary outcomes, \(R^2\) and standard residual plots aren’t particularly informative. Instead, we use specialized diagnostics. Let’s refit our Court models to demonstrate:

logitmod  <- glm(propetit ~ liberal2, family = binomial(link = logit), data = mydata)
logitmod2 <- update(logitmod, . ~ . + usparty + ineffcou + multpet)

y     <- mydata$propetit
pred1 <- predict(logitmod, type = "response")
pred2 <- predict(logitmod2, type = "response")

Hosmer-Lemeshow test

This test groups observations into deciles of predicted probability and compares observed vs. expected event counts. A significant result (\(p < 0.05\)) suggests poor fit:

library(ResourceSelection)
hoslem.test(mydata$propetit, fitted(logitmod))

ROC curves

A Receiver Operating Characteristic curve shows the trade-off between sensitivity (true positive rate) and specificity (1 - false positive rate) across all possible classification thresholds.

Using the ROCR package (with color for threshold values):

library(ROCR)

predRoc1 <- prediction(pred1, mydata$propetit)
perfRoc1 <- performance(predRoc1, "tpr", "fpr")
plot(perfRoc1, colorize = TRUE, main = "ROC Curve --- Bivariate Model")
predRoc2 <- prediction(pred2, mydata$propetit)
perfRoc2 <- performance(predRoc2, "tpr", "fpr")
plot(perfRoc2, colorize = TRUE, main = "ROC Curve --- Multivariate Model")

Or using the pROC package for a cleaner comparison:

library(pROC)
plot.roc(y, pred1, col = "red", main = "ROC Curves: Model 1 (red) vs. Model 2 (blue)")
plot.roc(y, pred2, col = "blue", add = TRUE)

Area Under the Curve (AUC)

The AUC summarizes the ROC curve in a single number. AUC = 0.5 means no better than chance; AUC = 1.0 means perfect discrimination:

auc(mydata$propetit, pred1)
Area under the curve: 0.7486
auc(mydata$propetit, pred2)
Area under the curve: 0.7901

Separation plots

Separation plots provide an intuitive visual assessment. Observations are sorted left-to-right by predicted probability. Each vertical line represents one case — dark lines are actual events (pro-petitioner decisions), light lines are non-events. A well-fitting model pushes all dark lines to the right:

library(separationplot)

separationplot(pred = logitmod$fitted.values, actual = logitmod$y,
               type = "rect", line = TRUE, show.expected = TRUE,
               width = 9, height = 1.2, heading = "Bivariate Model")

separationplot(pred = logitmod2$fitted.values, actual = logitmod2$y,
               type = "rect", line = TRUE, show.expected = TRUE,
               width = 9, height = 1.2, heading = "Multivariate Model")

For reference, here’s what a perfectly predicted model would look like — all events neatly stacked on the right:

e <- plogis(seq(-10, 3.02, length.out = 500))
f <- c(rep(0, 382), rep(1, 118))
separationplot(pred = e, actual = f, type = "rect", show.expected = TRUE,
               heading = "Perfect Prediction (Reference)")

Part 9: Interactions in Logit Models

Interactions work the same way conceptually as in OLS — the effect of one variable depends on the level of another. But because the logit link function is nonlinear, interpreting interactions requires extra care. Visualization is essential.

Liberalism x Party

Does the effect of the Court’s liberalism on petitioner outcomes depend on whether the U.S. government is a party to the case?

LPM vs. logit/probit OLS on binary outcomes produces impossible predictions; logit and probit constrain probabilities to [0, 1]
Interpretation Use odds ratios, predicted probabilities, or marginal effects — not raw coefficients
Logit vs. probit Nearly identical predicted probabilities; choice is largely a matter of convention
Diagnostics Hosmer-Lemeshow for calibration; ROC/AUC for discrimination; separation plots for visual assessment
Interactions Conceptually the same as OLS, but must visualize because the nonlinear link makes intuition harder
